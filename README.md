# Система аутентификации и авторизации

## Описание проекта
Backend-приложение, реализующее кастомную систему аутентификации и матричную систему разграничения прав доступа (RBAC). Проект разработан с использованием FastAPI и PostgreSQL, демонстрируя принципы Clean Architecture, безопасность на уровне JWT с возможностью отзыва токенов и гибкое управление правами доступа к бизнес-ресурсам.

## Архитектура системы доступа
Система основана на матрице прав, связывающей роли пользователей с конкретными бизнес-объектами.

### Структура базы данных
1. **users**: Хранит данные пользователей, хеши паролей и статус активности. Поддерживает мягкое удаление.
2. **roles**: Определяет роли в системе (Администратор, Менеджер, Пользователь).
3. **business_elements**: Описывает ресурсы приложения, к которым может быть ограничен доступ (например, товары, заказы).
4. **access_rules**: Матрица правил. Связывает роль и ресурс, определяя детальные права: чтение своих/чужих объектов, создание, обновление, удаление.
5. **user_sessions**: Таблица для управления активными сессиями. Позволяет реализовать полноценный выход из системы (logout) путем отзыва JWT-токенов до истечения их срока действия.

## Технологический стек
- **Язык**: Python 3.11+
- **Фреймворк**: FastAPI
- **База данных**: PostgreSQL
- **ORM**: SQLAlchemy 2.0 (Async)
- **Миграции**: Alembic
- **Безопасность**: PyJWT, bcrypt
- **Управление пакетами**: uv
- **Контейнеризация**: Docker, Docker Compose

## Локальный запуск (через uv)

Для запуска проекта на локальной машине выполните следующие шаги:

1. Установите зависимости:
```bash
uv sync
```

2. Активируйте виртуальное окружение:
   - **Linux / macOS**:
     ```bash
     source .venv/bin/activate
     ```
   - **Windows (PowerShell)**:
     ```powershell
     .venv\Scripts\Activate.ps1
     ```
   - **Windows (CMD)**:
     ```cmd
     .venv\Scripts\activate.bat
     ```
   - **Fish shell**:
     ```fish
     source .venv/bin/activate.fish
     ```

3. Запустите базу данных (если не используете Docker):
   Убедитесь, что PostgreSQL запущен и переменные `DATABASE_URL` и `SECRET_KEY` прописаны в файле `.env`.

4. Примените миграции базы данных:
```bash
uv run alembic upgrade head
```

5. Запустите сервер разработки:
```bash
uv run uvicorn src.main:app --reload --port 8001
```

Документация API будет доступна по адресу: `http://127.0.0.1:8001/docs`

## Запуск через Docker

Для полного развертывания приложения, включая автоматическое создание базы данных и применение всех миграций, используйте одну команду:

```bash
sudo docker compose up --build -d
```

Эта команда:
1. Соберет образ приложения.
2. Запустит контейнер с PostgreSQL.
3. Автоматически создаст все необходимые таблицы и начальные данные (сиды) при первом запуске.

Приложение станет доступно по адресу: `http://127.0.0.1:8001`
